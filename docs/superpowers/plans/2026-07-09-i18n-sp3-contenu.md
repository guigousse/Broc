# i18n SP3 — Contenu de jeu trilingue (FR/EN/ES) — Plan d'implémentation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Tout le contenu de jeu (objets, brocantes, compétences, déblocages, clients, vendeurs, expéditeurs, catégories, petits domaines) s'affiche dans la langue courante, résolu par id au rendu, avec fallback français.

**Architecture:** Le français dans `src/data/` reste canonique (zéro changement de structure ni de save). Les traductions vivent dans des overlays `src/lib/i18n/contenu/{en,es}/<domaine>.ts` (`Record<Id, string | {…}>`), consommés via des helpers de résolution dans `src/lib/i18n/contenu/index.ts` (`nomObjet`, `nomBrocante`, `nomCompetence`…). Chaque domaine a un test de complétude : chaque id FR doit avoir son entrée EN et ES, et aucune entrée orpheline.

**Tech Stack:** Next.js (export statique) + React, vitest, dictionnaires maison typés (SP1/SP2 déjà en place : `useLangue()` → `{ locale, d, tr }`).

## Global Constraints

- **Glossaire OBLIGATOIRE** : `docs/superpowers/specs/2026-07-08-i18n-glossaire.md` (chiner = pick / rebuscar, brocante = flea market / mercadillo, vide-grenier = yard sale / rastrillo, palier = tier / rango, etc.).
- **Règle d'or** : ne JAMAIS persister une chaîne localisée ; ne JAMAIS modifier une valeur de save ou une union interne (`CategorieObjet`, `EtatObjet`, ambiance de brocante…). Résolution à l'affichage uniquement, fallback FR si entrée absente.
- **Ne JAMAIS renommer un `templateId`** (règle post-migration `templateIdRenames.ts`).
- **Noms-parodies** (droits d'auteur, ton validé) : garder le clin d'œil RECONNAISSABLE dans chaque langue — adapter le jeu de mots, jamais le nom déposé réel, jamais de version fade. Ex. « Vinyle des Scarabées — 'Passage Clouté' » → EN « The Scarabs' record — 'Zebra Crossing' » (clin d'œil Abbey Road). Domaine public (Hugo, Voltaire, Paganini…) : noms réels conservés, titres d'œuvres dans leur forme usuelle par langue (« Les Misérables » reste tel quel en EN, « Los Miserables » en ES).
- **Registre** : EN anglais international, ES d'Espagne neutre, ton chaleureux vintage France 1924, tutoiement ES « tú », prix toujours en €.
- **Longueur** : l'ES est ~+10 % plus long que le FR — rester compact sur les noms affichés en badge/ligne.
- Tests : `npx vitest run <fichier>` ciblé, `npm run test:run` complet. Lint : `npx eslint src` (`npm run lint` est cassé, Next 16).
- Zéro dépendance npm nouvelle.
- **Hors périmètre SP3 (= SP4, ne pas toucher)** : `Courrier.payload.titre/corps` (lettres et quêtes persistées), gabarits de quêtes générées (`src/lib/quetes/textes.ts`), répliques de négociation (`NegociationState.message`), `LedgerEntry.designation`, textes générés de la gazette.

```ts
// src/lib/i18n/contenu/index.ts
import type { Locale } from "@/lib/i18n/locales";
import { getTemplate } from "@/data/objetTemplates";
import { OBJETS_EN } from "./en/objets";
import { OBJETS_ES } from "./es/objets";

/**
 * Overlays de contenu (spec i18n §2) : le français de `src/data/` est
 * canonique, EN/ES sont des Record<Id, …> résolus À L'AFFICHAGE.
 * Fallback FR si entrée absente — jamais de crash, jamais d'écriture en save.
 */
const OBJETS: Record<"en" | "es", Record<string, string>> = {
  en: OBJETS_EN,
  es: OBJETS_ES,
};

/** Nom localisé d'un template d'objet. Id inconnu → id brut (marqueur repérable). */
export function nomTemplate(templateId: string, locale: Locale): string {
  const fr = getTemplate(templateId)?.nom;
  if (locale !== "fr") {
    const trad = OBJETS[locale][templateId];
    if (trad) return trad;
  }
  return fr ?? templateId;
}

/**
 * Nom localisé d'un objet/slot possédé. Le champ `nom` persisté est un
 * snapshot FR hérité (règle d'or : plus jamais affiché tel quel) — on résout
 * par templateId ; le snapshot ne sert que de repli (templateId "legacy").
 */
export function nomObjet(
  o: { templateId: string; nom: string },
  locale: Locale,
): string {
  if (locale !== "fr") {
    const trad = OBJETS[locale][o.templateId];
    if (trad) return trad;
  }
  return getTemplate(o.templateId)?.nom ?? o.nom;
}

/** Utils des tests de complétude par domaine. */
export function manquants<T>(ids: string[], overlay: Record<string, T>): string[] {
  return ids.filter((id) => !(id in overlay));
}
export function orphelins<T>(ids: string[], overlay: Record<string, T>): string[] {
  const connus = new Set(ids);
  return Object.keys(overlay).filter((k) => !connus.has(k));
}
```
